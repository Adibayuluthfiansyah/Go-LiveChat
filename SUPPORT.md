# Support

RAWVE is an early-stage open source project maintained by one person.
There's no support team — just a developer who wants to build something better for creators.
Here's how to get help, and how to help back.

---

## Read the docs first

The [README](README.md) covers installation, environment setup, and architecture.
Most questions at this stage are answered there.

---

## Found a bug or have a feature request?

Open a GitHub Issue — it's the main way to track problems and ideas.

**How to open an issue:**

1. Go to: https://github.com/adibayuluthfiansyah/RAWVE-LiveStream-Platform/issues
2. Click **"New issue"**
3. Choose a title that describes the problem clearly
4. Describe what you expected vs what happened (for bugs), or why the feature matters (for requests)
5. Include your Go version, OS, and any relevant error output if reporting a bug

There are no issue templates yet — just write naturally. Every issue helps, even if it's just a question.

> This project has no contributors yet. If you're the first to open an issue, that's genuinely appreciated.

---

## Want to ask something less formal?

If it's more of a question than a bug report, open a **GitHub Discussion** instead:

https://github.com/adibayuluthfiansyah/RAWVE-LiveStream-Platform/discussions

Discussions are good for things like:
- "Is this the right way to approach X?"
- "I want to contribute but don't know where to start"
- "What's the plan for feature Y?"
